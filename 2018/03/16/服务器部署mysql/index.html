如何在服务器上面部署MYSQL

后端 分享 2018/03/16

1. 下载压缩包

mysql-5.5.59-linux-glibc2.12-x86_64.tar.gz

2. 解压

tar -zxvf mysql-5.5.59-linux-glibc2.12-x86_64.tar.gz

3. 将文件复制到 /usr/local/ 并命名为mysql文件

cp mysql-5.5.59-linux-glibc2.12-x86_64 /usr/local/mysql

4. 查看profile文件，并配置mysql环境变量

vi /etc/profile

5. 应用

source /etc/profile

6. 增加用户组

groupadd mysql
useradd mysql:mysql

查看所有的组

cat /etc/group

查询mysql用户的所在的组

groups mysql

修改组

groupmod -n test1 test
将test组的名子改成test1

删除组

groupdel test

linux下添加，删除，修改，查看用户和用户组

7. 保存 basedir datadir 等信息

cd /etc/my.cnf

直接加入下面的信息

给mysql用户组下面的mysql用户权限访问data的权限，或者将data的权限调低

chown -R mysql:mysql /data
chmod -R 755 /usr/local/mysql/data

// 如果没有文件的话就新建mariadb.log,mariadb.pid

可以将mysql.server 复制到/etc/init.d/mysql ,目的设置开机启动
也可以通过 /etc/init.d/mysql start 启动

8. 启动mysql

在support-files 里面 ./mysql.server start

启动的时候一直报异常，根据网上找了一圈，感觉把所有的方法都试一下遍

[root@snsgou mysql]# service mysql restart
MySQL server PID file could not be found!
Starting MySQL…The server quit without updating PID file (/usr/local/mysql/data/*.pid)

最后解决是这个语句 mv /etc/my.cnf /etc/my.cnf.backup 然后启动服务器就可以了
好像说的是版本的不兼容这个的问题
