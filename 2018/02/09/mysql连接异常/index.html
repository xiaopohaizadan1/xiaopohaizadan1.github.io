MYSQL 连接异常

后端 java8 2018/02/09 阅读数: 1000

com.mysql.jdbc.exceptions.jdbc4.CommunicationsException: Communications link failure

记录下自己折腾的过程 百度了下这个异常说的全部是下面的解决办法，但是并没有解决我问题：

1.数据库连接的问题

数据库连接池问题？没有设置数据库连接池，所以整个项目只有一个连接,也没有设置数据库的超时时间，导致连接其实被mysql已经断开了，但是项目的中的这个连接以为自己还处于连接状态，实际是已经断开的，所以连接的时候一直连接不上，报了这个异常;

网上找的方法：

a.my.conf 里面修改【mysqld】等待时间 和 有效连接时间

然后重启mysql 发现并没有用

b.在请求url中加入 autoConnection = ture

也是并没有用

2. 是不是nginx代理问题？

关闭代理之后在连接数据库发现还是不行~~~~


这个问题感觉是无解的~~，然后迫不得已的情况下，删除了msyql，重新安装~~~然后第二次安装mysql也是各种的坑~~蛋疼~
