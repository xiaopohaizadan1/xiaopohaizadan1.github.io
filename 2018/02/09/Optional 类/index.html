Optional 类

后端 java8 2018/02/09 
一个属性
private final T value; // 最终对于其他方法的判断都是以这个值来进行判断的

private static final Optional<?> EMPTY = new Optional<>();


Optional无参的构造方法将value值赋值为null

然鹅这个有参的构造方法要求的值必须是非空的
private Optional(T value) {
    this.value = Objects.requireNonNull(value);
}

Optional.of()的参数必须是非空的
这个方法返回的式Optional的一个有参的构造方法


Objects.requireNonNull(); 这个方法的参数如果是空的则会抛出空指针异常

ofNullable( T value) // 其实这边就是将可能为空的一个用一个无参的Optional的对象来替换 将value设置为null     如果不为空的值就用有参数的Optional对象来替换  将value设置为  ofNullable( T value) 方法的值
value == null ? empty() : of(value)

isPresent() // 就是判断value的值是不是为空  空返回false 非空的 返回true
