  1. from types import MethodType

    def  set_age(self,age):

self.age = age

class Student(object):

 pass

 c = Student()

     set_age = MethodType(set_age,c)

 set_age(12)

 c2 =Student()

 c2.set_age(12)  会报错

  2. __slots__ = (‘name’,’age’)

    限制只能使用这个两个属性，类中定义的

    如果这个类的实例可以使用，子类是不限制的(__slots__定义的属性仅对当前类实例起作用，对继承的子类是不起作用的：)

  1. @property 把一个方法变成属性来调用

    @property
def foo(self):
    return self._foo

@foo.setter
def foo(self,foo):
    self.__foo = foo

可以foo.setter 装饰的方法和property修改的名称不一样  但是在作为属性调用的时候要按照方法名称来，不然会报错

@property是不是只能作用于get的方法？ 结果测的好像确实如此

  1. 只读属性

    只有getter方法取出被配置了，没有setter方法

定制类

\_\_init__()          初始化

\_\_str__()         print(实例对象的时候打印的)

\_\_repr__()    直接打印实例对象的时候显示的

\_\_iter__()        类想要被作用于for。。 in 循环

\_\_getitem__()

\_\_setitem__() 

\_\_delitem__()

\_\_getattr__()  只有在没有找到属性的时候才会调用该方法查找属性
