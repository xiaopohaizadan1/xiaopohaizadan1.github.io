java 8

1.lambda表达式
闭包，推动java8发布的最重要的新特性
函数作为参数传递进方法

lambda表达式的重要特性
1.可选类型声明
    不需要声明参数类型，编译器可以统一识别参数值

    // 类型声明
  MathOperation addition = (int a, int b) -> a + b;

      // 不用类型声明
  MathOperation subtraction = (a, b) -> a - b;


2.可选的参数圆括号
    一个参数无需定义圆括号，多个参数需要定义
    GreetingService greet = message -> "哈哈";

    // 多个参数必须要有圆括号，一个也可以写
    GreetingService greet = (message) -> "哈哈";


3.可选的大括号
    如果主体包含一个语句就不需要使用

    如果接口的方法的是没有返回值的话，加不加大括号都是可以的，但是加大括号一定要注意分号

    GreetingService2 greet = message -> System.out.println(message + "");

    GreetingService2 greet = message -> {System.out.println(message+"");};

4.可选的返回关键字
    如果主体只有一个表达式返回值，则编译器会自动返回值，如果有多个返回值值需要指明返回值

    MathOperation addition = (a,b) -> {a + b};
    // ❎ 在大括号里面默认的认为你有多条返回值，没有明确的返回值

    MathOperation addition = (a,b) -> {return a + b;}; 
    // ✅ 大括号括起来的必须要有分号

// 接口
interface MathOperation {
int Operation(int a, int b);
}

// 接口
interface GreetingService {
    String sayMessage(String message); 
}

// 接口
interface GreetingService2 {
    void sayMessage(String message); 
}


Not a statement        

missing return statement

Multiple non-overriding abstract methods found in interface
// 针对于GreetingService接口来说，message是GreetingService接口里面一个方法的参数
// 类型可以不用声明出来如果系统可以自动识别到对应的式这个方法
// "->"后面的值是这个方法返回的类型

lambda表达式主要用来定义行内执行的方法类型接口？不理解
eg. 一个简单的方法接口 使用lambda表达式来定义 接口的方法 (是不是相当于lambda表达式具体的来定义了接口的方法的实现)

免去了使用匿名方法的麻烦，并且给予Java简单但是强大的函数化的编程能力。

MathOperation addition = (a,b) -> {a + b};

MathOperation  addition = new MathOperation(){
    @override
    public int Operation(int a ,int b){
        return a+b;
    }
}

2.方法的引用
​
凡是使用lambda表达式的地方都可以使用方法的引用
// 构造器的引用 等价于
Car car = Car.create(Car::new);
Car ca1 = Car.create(() -> new Car());

// 静态方法  静态有参的
List cars = Arrays.asList(car); // 参数必须是Car类型的 不然编译的时候会报错，猜测应该是不知道该怎么使用集合里面的car1实例

cars.forEach(Car::repair);
cars.forEach(car1 -> Car.repair(car1)); 

// 非静态的方法 无参的 特定类的任意对象的使用
cars.forEach(Car::collide);
cars.forEach(car1 -> car1.collide());

// 特定对象的使用
cars.forEach(car::follow)
cars.forEach(car1->car.follow(car1));


通过方法的名字来指向一个方法 (只能使用于当前类的无参的方法，有参数的是不是必须参数是当前类的方法才可以引用)
:: 使用两个冒号来操作

Car::new 构造器方法的引用

Car::repar 后面的方法要不是静态的要不就是无参数的

如果是有参数的是需要针对于具体的Car的实例来引用方法

non-static method cannot be referenced from a static context

3.函数式接口 @FunctionalInterface
1.有且仅有一个抽象方法
2.default 和static 修饰的方法 不算抽象方法
3.java.lang.Object里面的方法也都不是抽象方法

常见的函数式接口:
1.Consumer<T> 接受一个输入参数并且无返回
2.Predicate<T> 接受一个输入参数返回boolean值
3.Supplier<T>  无参数返回一个结果

4.默认方法
接口的默认的方法:目的解决接口的修改与现有的接口实现不兼容的问题

如果一个类实现的两个接口中有同名的默认方法，则必须要实现该方法，不然编译报错。 
同名的静态默认方法和默认方法不影响。同名的静态默认方法也不影响

可以通过接口名.super.方法名来实现具体的接口的某个默认方法

接口可以声明静态方法（只需要加static修饰）

5.Stream 流
+——————–+ +——+ +——+ +—+ +——-+
| stream of elements +—–> |filter+-> |sorted+-> |map+-> |collect|
+——————–+ +——+ +——+ +—+ +——-+

什么是流?

来自数据源的元素队列并支持聚合操作

数据流:流的来源 集合，数组，I/O channel，产生器generator

元素队列：特定类型的对象形成的队列

聚合操作：filter，map，reduce。。。。

Stream还有两个特性

1、pipelinling(管道)

2、内部迭代 —–访问者模式(Visitor)实现

集合的接口有两种方式生成流

1.stream()

​ a.foreach 迭代流中的元素

​ 集合的map方法参数是一个实现Function接口的函数

​ 函数里面的抽象方法是传入一个任意类型的参数，返回一个任意类型的参数

​ b.map 映射每个元素对应的结果

​ 1.IntStream，doublieStream，Stream对应的map方法的参数都不一样

​ 2.每个Stream之间都是可以相互转化的。通过mapTo**()的方式

​ c.filter 过滤出满足条件的元素

​ d.limit 取多少个元素

​ e.sorted 排序

2.parallelStream() 流的并行处理程序的替代的方法

惰性求值和及早求值 只需要看返回值，返回值是Stream的就是惰性求值(只描述Stream，不产生新的集合)，返回值是其他的值或空的就是及早求值。

Collection的类的方法：

Collectors 实现归约的操作，例如将流转换成集合和聚合元素。Collectors 可用于返回列表或字符串

Collectors.toList()
